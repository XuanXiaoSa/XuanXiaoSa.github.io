git的使用

将远程项目克隆到本地

  • git clone git@202.107.200.26:xuanxiao/myOnehome.git
  • cd myOnehome // 进入该项目
  • git branch // 查看当前的分支 默认是master分支
  • git fetch origin develop:develop // 拉取远程分支都本地 任意远程已经有的分支
  • git branch // 现在应该有了两个分支 master和develop
  • git checkout develop // 可以切换到develop分支上

git的一些基本命令的使用

  • git clone git@202.107.200.26:xuanxiao/myOnehome.git
  • cd myOnehome // 进入该项目
  • git status // 查看当前文件暂存区的文件修改状态
  • git branch -r // 查看远程分支
  • git branch // 查看当前的分支
  • git branch develop // 新建一个develope分支
  • git checkout develop // 切换到develop分支
  • git add ./README.md // 添加所有的文件后者 readme.md文件
  • git commit -m ‘这里写备注’ // 提交修改的代码到暂存区，书写备注
  • git push // 将当前分支上传到远程对应的分支
  • git pull // 将当前分支对应的远程分支拉取到本地
  • git merge master // 将master分支上面的代码和合并到develop分支上
  • git push –set-upstream origin feature // 将当前分支推到远程，远程没该分支会新建
  • git branch -d feature //删除feature分支 这个删不删在于你哟，分支没用了删除会显瘦哦
  • git branch -u upstream/foo foo // 建立映射
  • git branch –set-upstream-to=origin/dev dev // 建立映射

代码回滚

  • git log // 查看提交版本日志 有个commit 后面的字符串就是回滚id

  • git reset –hard HEAD^ // HEAD是当前提交版本 HEAD^是上一个提交版本 多个(如10个)的话可以使用 HEAD~10 也可以不使用 HEAD 直接使用回滚id

  • git reset –hard ef5f9807 // 拿到回滚之前的commit id 可以会滚到回滚之前的版本

  • git checkout – readme.txt // 可以将该文件的所有修改恢复到出事状态

创建标签 tag

  • git tag v1.0
  • git tag 查看当前的tag
